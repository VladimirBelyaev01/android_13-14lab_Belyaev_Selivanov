# Unscramble Game

## Описание игры
Игра-головоломка, в которой игроку необходимо угадать зашифрованное слово, переставляя буквы в правильном порядке. Игра помогает развивать словарный запас и навыки составления слов.

## Технологии
- Kotlin
- Jetpack Compose
- ViewModel
- StateFlow

## Функциональность приложения
- Отображение перемешанного слова
- Поле ввода для ответа
- Проверка правильности введённого слова
- Подсчёт очков
- Отслеживание прогресса (номер текущего слова)
- Переход к следующему слову
- Сброс игры
- Сохранение состояния при повороте экрана

## Контрольные вопросы

### 1. Что такое ViewModel и зачем он нужен?
ViewModel — компонент архитектуры Android, который хранит и управляет данными UI, переживая изменения конфигурации (поворот экрана). Без ViewModel при повороте экрана Activity пересоздаётся, и все данные теряются.

### 2. В чём разница между MutableStateFlow и StateFlow?
MutableStateFlow — изменяемый поток, StateFlow — только для чтения. Два поля (_uiState и uiState) используются для паттерна "backing property": _uiState доступен только внутри ViewModel для изменения, uiState доступен снаружи только для чтения.

### 3. Почему в ViewModel нельзя хранить Context или View?
Это приводит к утечкам памяти, так как ViewModel живёт дольше Activity. Сохранённая ссылка не даёт Activity удалиться сборщиком мусора.

### 4. Чем StateFlow отличается от remember и rememberSaveable?
StateFlow живёт в ViewModel и переживает повороты экрана, remember хранит состояние только в рамках композиции, rememberSaveable сохраняет состояние через Bundle при повороте.

### 5. Что такое data class в Kotlin и зачем он используется для UI State?
data class автоматически генерирует equals(), hashCode(), toString(), copy(). copy() позволяет создавать копии с изменением отдельных полей, что удобно для неизменяемых UI State.

### 6. Как работает collectAsState() в Compose?
collectAsState() подписывается на StateFlow и при каждом изменении значения вызывает перекомпозицию UI, автоматически обновляя интерфейс.

### 7. Зачем разделять код на пакеты data и ui?
Это поддерживает принципы чистой архитектуры: разделение ответственности, тестируемость и независимость слоёв.

### 8. Что происходит с ViewModel, когда приложение полностью закрывается?
ViewModel уничтожается, данные теряются. Для долговременного хранения используют DataStore, Room, SharedPreferences.
